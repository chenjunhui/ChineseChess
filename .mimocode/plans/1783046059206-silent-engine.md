# 旁观者功能实现计划

## 概述
在每张桌子上添加"旁观"按钮，允许玩家以旁观者身份进入正在对弈的棋局，只能观战不能操作棋子，旁观人数不限。

## 设计思路

当前广播消息通过 `clientInfo.tableId === client.tableId` 过滤接收者。旁观者只需设置 `client.tableId` 即可自动收到走棋消息。主要需要处理的是：旁观者进入时发送当前棋盘状态、游戏重启时同步旁观者、以及客户端禁止旁观者操作。

## 修改文件清单

### 1. `server/protocol.js` — 新增消息类型
```
+ TABLE_WATCH: 'table.watch'
```

### 2. `server/rooms.js` — 桌子增加旁观者追踪
- 每个 table 对象新增 `spectators: []`（存储 `{userId, playerName}`）
- 新增 `addSpectator(tableId, userId, playerName)` — 添加旁观者，返回 `{success}`
- 新增 `removeSpectator(tableId, userId)` — 移除旁观者，返回 boolean
- 更新 `getTables()` 返回值增加 `spectatorCount: table.spectators.length`

### 3. `server/index.js` — 核心逻辑

**新增 `TABLE_WATCH` handler：**
1. 校验：桌子存在且 `table.isPlaying` 为 true
2. 调用 `addSpectator(tableId, userId, playerName)`
3. 设置 `client.tableId = tableId`（使现有广播逻辑自动到达旁观者）
4. 设置 `client.isSpectator = true`（旁观者标记）
5. 查找当前 GameState，发送 `GAME_START` 给旁观者：
   ```js
   send(ws, MSG.GAME_START, {
     tableId, board: game.board,
     yourColor: null,  // 旁观者无颜色
     opponentName: seat0.playerName + ' vs ' + seat1.playerName,
     gameMode: table.gameMode,
     isSpectator: true
   })
   ```
6. 广播 `LOBBY_UPDATE` 更新旁观者人数

**修改 `startGameForTable`（lines 53-66）：**
- 现有循环只处理玩家，对 `clientInfo.isSpectator` 的客户端单独处理：
  ```js
  if (clientInfo.isSpectator && clientInfo.tableId === tableId) {
    send(clientWs, MSG.GAME_START, {
      tableId, board: gameState.board,
      yourColor: null,
      opponentName: seat0.playerName + ' vs ' + seat1.playerName,
      gameMode: mode, isSpectator: true
    })
  }
  ```

**修改 `TABLE_LEAVE`：**
- 旁观者离开时（`client.isSpectator`）：调用 `removeSpectator()`，清除 `client.tableId` 和 `client.isSpectator`，广播 `LOBBY_UPDATE`

**修改 `ws.on('close')`：**
- 断线时检查并清理旁观者：调用 `removeSpectator()`，广播 `LOBBY_UPDATE`

## 数据流

```
用户点击旁观 → TableCard emit 'watch'
  → LobbyView.onWatch → lobby.watch(tableId)
  → 发送 TABLE_WATCH 给服务端
  → 服务端: addSpectator + client.tableId = tableId + client.isSpectator = true
  → 服务端发送 GAME_START (isSpectator: true, 当前 board)
  → 客户端: game.startGame(payload) → 跳转 /game
  → isSpectator = true，禁止所有操作
  → 走棋消息通过现有广播自动到达旁观者
```

## 验证方式
1. 启动前后端服务
2. 用户 A 坐下开始对弈
3. 用户 B 在大厅看到"旁观"按钮，点击进入
4. 用户 B 显示"旁观中"标签，无法操作棋子
5. 用户 A 走棋，用户 B 实时看到更新
6. 用户 B 点"退出旁观"回到大厅，大厅显示旁观人数
7. 多个用户可同时旁观同一局
8. 旁观者断线不影响对局
